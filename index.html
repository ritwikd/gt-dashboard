    Welcome to the project page for the Georgia Tech Health Dashboard!

    This is a self-contained health dashboard software system that provides customizable data tracking for long-term-care patients.

    You can download a release copy of the project and its source code here.

    This software is provided as-is. If any bugs are found, please feel free to submit a properly-tagged issue in the GitHub tracker. If you want the bug to be fixed quickly, fork the repo, find the bug's location in the code, and put the info into the issue, or just submit a pull request, and I'll merge it in.

    Note: Using this project requires basic familiarity with the Linux command line. You should be able to comfortably navigate the directory structure and run executables with elevated privileges as necessary. You should also be able to close processes (I recommend using htop).

    Getting Started

    Download and extract the project archive.

    Go to the gt-dashboard directory.

    The application consists of a several parts:

    • A portable nginx server (version 1.6.3)
    • A portable mongo database (version 2.6.3)
    • A Flask server (version 0.10.1)

    All of these are started and stopped by the "run" file in the root directory.

    To start the various applications, type:

    user :: ~/gt-dashboard <master)> >> ./run start

    To stop everything (additional checking may be required), type:

    user :: ~/gt-dashboard <master)> >> ./run stop
